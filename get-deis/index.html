Deploy Your Own Private PaaS

Before you get started, read about Deis core Concepts. Review our documentation for more detailed instructions, and see the README for installation prerequisites.

Want help? OpDemand provides installation assistance and commercial support.

01. Provision Deis

{% highlight bash %} $ curl -sSL http://deis.io/deisctl/install.sh | sh $ sudo mv deisctl /usr/local/bin $ git clone https://github.com/deis/deis.git $ export DEIS_NUM_INSTANCES=3 $ cd deis $ make discovery-url && vagrant up $ ssh-add ~/.vagrant.d/insecure_private_key $ export DEISCTL_TUNNEL=172.17.8.100 $ deisctl config platform set \ domain=local3.deisapp.com \ sshPrivateKey=~/.vagrant.d/insecure_private_key $ deisctl install platform && deisctl start platform $ pip install ./client/ $ deis register http://deis.local3.deisapp.com $ deis keys:add {% endhighlight %}

Provision a Deis cluster. Here we use Vagrant, but Deis also runs on Rackspace, Amazon AWS, DigitalOcean, Google Compute Engine, and bare metal.

Use the Deis command-line client to interact with the controller.

Use the Deisctl client to manage a Deis cluster.
